## Abstract

We validated the synthesis pipeline against five corpora with **known
ground-truth structure**, to test whether it (a) merges paraphrases of one idea
into one *synth*, (b) keeps distinct-but-close ideas separate, (c) keeps opposite
stances separate, (d) leaves a lone outlier unclustered, and (e) groups synths
into the correct topics. Running these in dev surfaced **three real production
bugs**, all now fixed with regression tests. After the fixes, the mechanism
recovers ground-truth **synth** structure exactly on realistically-worded
corpora, correctly separates negated opposites, and leaves lone outliers
unclustered. The dominant residual failure mode is **corpus-realism**: when
synthetic paraphrases are written with maximal lexical variation (unrealistically
low within-synth cosine), the medoid-anchored judge fragments synths. Topic-level
grouping via the diagnostic single-linkage path is recovered only for
clearly-distinct topics, not closely-related sub-topics; it is not a production
code path.

## 1. Objective & method

Each run: author a corpus with labelled ground truth → seed as raw options in the
Firestore emulator with live synthesis OFF (embeddings generated by the standard
option-creation trigger) → run the **production** bulk path
(`bulkClusterByEmbedding` UMAP→DBSCAN, then `twoTierJudge`: medoid cosine-bands +
LLM semantic-equivalence + complete-linkage `refineComponent`) → export artifacts
→ score with a structure-agnostic scorer (`score.mjs`) that derives the expected
structure from the per-statement labels. Protocols:
`scientific-research/DESIGNING-TEST-CORPORA.md` and
`PREPARING-VALIDATION-REPORTS.md`. Each run has its own `report.md` with full
parameters, sweep, and reproduction commands.

**Disclosure (threat to validity, applies to every run):** all corpora are
**LLM-authored synthetic** sentences (Claude), not human deliberation. They are
cleaner and more on-message than real input. A pass here is *necessary, not
sufficient*; robustness must be assessed on production-derived corpora.

## 2. The corpora

| Run folder | Structure | Archetype | Tests |
|---|---|---|---|
| `1-6-2026-40-20-10-validation` | 2 topics × 2 synths × 10 = 40 | A — clean recovery (reference) | synth + topic recovery |
| `4-6-2026-60-20-5-validation` | 3 topics × 4 synths × 5 = 60, **loose** wording | A — finer, low-cosine | recovery under low within-synth cosine |
| `4-6-2026-60-20-5-tight-validation` | 3 topics × 4 synths × 5 = 60, **tight** wording | A — realistic | recovery under realistic cosine |
| `4-6-2026-20-10-5-negation-validation` | 2 subjects × 2 stances × 5 = 20 | E — opposites | negation/stance must not merge |
| `4-6-2026-41-20-4-singleton-validation` | 2 topics × 5 synths × 4 = 40 + 1 outside = 41 | C/F — lone statement | outlier must stay unclustered |

"Loose" vs "tight" differ only in **within-synth wording width** (how similar the
5 paraphrases of one idea are), which sets their cosine to the cluster medoid:

| | within-synth cosine (min–mean–max) | clears 0.94 auto-accept? |
|---|---|---|
| loose | 0.727 – 0.816 – 0.896 | no (0/120 pairs) |
| tight | 0.933 – 0.972 – 0.997 | yes (118/120 pairs) |

## 3. Results

Scored by `score.mjs` (Level 1) and, for the deterministic clustering step,
re-derived from shipped embeddings by `verifyFromEmbeddings.ts` (Level 2).

| Run | score | Verdict |
|---|---|---|
| reference (40-20-10) | 10/10 | **PASS** — 4 pure synths, 2 correct topics, 0 overlap |
| loose (60-20-5) | 9/19 | **FAIL (partial)** — `eat-vegetables`/`portion-control` correctly split (the judge's core job), but 6/12 synths fragmented, 47/60 assigned |
| tight (60-20-5) | 17/19 | **PASS (synth)** — **12/12 pure synths, 60/60 assigned, 0 overlap, 0 LLM calls**; 2 fails are the diagnostic topic grouping |
| negation (20-10-5) | 7/11 | **PASS (property)** — **4 stance-pure synths, opposites not merged, 0 cross-stance contamination, 2 correct topics**; 4 fails are recall (3/20 dropped) |
| singleton (41-20-4) | 18/18 | **PASS** — 10/10 pure synths, 2 correct topics, **off-topic statement left unclustered** |

Headline contrasts:

- **Loose vs tight** (identical structure, different wording width): synths
  recovered pure went **6/12 → 12/12**, options assigned **47/60 → 60/60**, judge
  LLM calls **63 → 0**. The loose run's fragmentation was therefore a
  corpus-realism artifact, not a mechanism defect — confirmed by holding the
  structure fixed and only tightening the wording.
- **Negation**: a cosine-only path returns 2 synths (pro+anti conflated per
  subject); the production two-tier judge returns **4 stance-pure synths**. The
  LLM verifier is what separates "make peace" from "do **not** make peace."
- **Singleton**: the cosine step groups the off-topic statement into the nearest
  cluster (UMAP proximity), but the judge now **excludes** it → unclustered.

## 4. Production bugs found and fixed

Validating in dev surfaced three real defects in the production primitives. Each
is fixed with a regression test; none is a workaround.

| # | Bug | Symptom | Fix | Commit |
|---|---|---|---|---|
| 1 | **DBSCAN border double-claim** — `density-clustering` places a border point in two clusters and leaves cluster members in its `noise` array, which the noise-reassignment step re-adds | 60 options → 66 memberships at ε=0.30 (non-disjoint) | `partitionDisjoint` (nearest-centroid dedupe) + skip already-assigned indices in noise reassignment. No-op at the production default ε=1.0 | `7465c9342` |
| 2 | **Over-high `autoRejectBand` (0.82)** — sat inside the within-synth cosine distribution (dips to 0.73), so valid paraphrases were demoted to dissent **without an LLM call** | clusters collapsed/fragmented on low-cosine corpora; valid members silently dropped | lower the default to **0.60** (a "clearly unrelated" floor); the LLM arbitrates the overlap zone. Bounded by the call cap | `7465c9342` |
| 3 | **Keep-branch retained dissenters** — at ≥80% agreement the synth was set to the *entire* cluster, including cosine-auto-rejected / LLM-dissenting members | a lone off-topic statement absorbed into an otherwise-coherent synth | a kept synth always contains **only the agreed subset**; dissenters are excluded (refined out or left unclustered). Coherent clusters unchanged | `88cab6a16` |

All three live in shared primitives, so **both** the live and bulk production
paths inherit the fixes (`VerifiedCluster.memberIds`, `bulkClusterByEmbedding`).

## 5. Conclusions

1. **Synth recovery is correct and trustworthy** on realistically-worded input:
   exact recovery (12/12 pure, full coverage, zero overlap) on the tight corpus,
   with the judge making zero LLM calls because the cosine tier handles
   high-similarity members.
2. **The LLM equivalence judge does what cosine cannot**: it separates
   distinct-but-close ideas (`eat-vegetables` vs `portion-control`) and opposite
   stances (`make-peace` vs `no-peace`) that sit at high cosine.
3. **Lone outliers are left unclustered** after fix #3, even when UMAP/DBSCAN
   groups them into a nearby cluster.
4. **Corpus realism is the dominant variable.** Maximal lexical variation pushes
   within-synth cosine below the auto-accept band, forcing every member through
   the (imperfect, medoid-anchored) LLM judge and causing fragmentation. Real
   human near-duplicates share more wording and would clear the band.
5. **Two parameters are corpus-dependent and must be reported, not silently
   tuned**: DBSCAN `ε` (synth vs topic granularity; the recovery window is
   narrow and corpus-specific) and the LLM-call cap (the production
   `min(2000, ⌈N×0.2⌉)` starves a small validation corpus and was lifted, with
   disclosure, for these runs).

## 6. Threats to validity & limitations

- **Synthetic corpora** (see Disclosure). Necessary-not-sufficient evidence.
- **Topic-level grouping is unproven for closely-related topics.** The
  single-linkage centroid agglomeration in `bulkRebuild.ts` is a *diagnostic*,
  not a production path; it recovered clearly-distinct topics (fitness vs
  finance; peace vs taxes) but chained closely-related sub-topics (three health
  sub-topics). Production forms topics via the live LLM-judged band router, which
  these runs did not exercise. **This is the main open item.**
- **Medoid-anchored recall.** Even with realistic cosine, members judged only
  against a single medoid can be dropped to dissent (e.g. the negation run shed
  3/20). Anchoring judgement to multiple references is a candidate improvement.
- **Determinism scope.** Cluster *membership* (UMAP seed 42, DBSCAN ε) is
  deterministic and re-derivable offline from the shipped embeddings. The
  `twoTierJudge` split, synth/topic *titles*, and which gray-band members are
  kept are **LLM-driven and not deterministic**. Embeddings depend on
  `text-embedding-3-small`; the input vectors are shipped so the clustering step
  replicates without the embedding API.

## 7. Reproducibility

Three levels per run (cheapest first); exact commands in each run's `report.md`.

1. **Verify the verdict from committed artifacts** (no deps/keys/emulator):
   `node score.mjs <run-folder>`.
2. **Re-derive the clustering from shipped embeddings** (no OpenAI/emulator):
   `cd functions && npx tsx scripts/verifyFromEmbeddings.ts ../scientific-research/validation/<run-folder> --eps=<ε> --seed=42`.
   Reproduces the deterministic membership; the LLM judge step is not replayed
   offline.
3. **Full end-to-end** (emulator + API keys): seed corpus, then
   `bulkRebuild.ts <qid> --eps=<ε> --two-tier --max-llm-calls=2000 --execute`.

Corpora: `scripts/seedSynthBenchmark.{healthy,healthyTight,negation,singleton3}.json`.
