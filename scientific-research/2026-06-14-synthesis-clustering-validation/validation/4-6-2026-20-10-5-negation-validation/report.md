## 1. Objective

Test that **statements which are semantically near but opposite in stance**
(negations) are **not merged** into one synth. This is archetype E in
`DESIGNING-TEST-CORPORA.md`, which warns that the cosine-only path *will* merge
opposites (they sit at high cosine) and that splitting them requires the LLM
verifier — i.e. the **two-tier judge**. This run validates that the production
judge does the split.

## 2. Dataset

```
2 subjects × 2 opposing stances × 5 paraphrases = 20 options
```

| Subject | Synth (stance) | role |
|---|---|---|
| peace | make-peace ("We want to make peace…") | — |
| peace | no-peace ("We do **not** want to make peace…") | opposite |
| taxes | raise-taxes ("We should raise taxes…") | — |
| taxes | no-raise-taxes ("We should **not** raise taxes…") | opposite |

Each stance-synth's 5 paraphrases are tightly worded; the pro/anti pair on a
subject differs essentially by negation. LLM-authored synthetic (Claude),
2026-06-04. Labels in [`statements.json`](./statements.json).

## 3. Procedure & parameters

Seed (synthesis OFF, 20/20 embeddings) → `bulkClusterByEmbedding` (ε=0.8, seed
42) → `twoTierJudge` → write synth docs. ε=0.8 is required here: at ε=0.45 the
20 near-similar statements formed **no** dense cluster (all noise); ε≥0.8 forms
the per-subject clusters (a corpus-dependent finding — the working set is small
and semantically tight). `autoRejectBand` 0.60, `autoAcceptBand` 0.94,
`maxLlmCalls` 2000. Full manifest in [`results.json`](./results.json).

**The cosine step merges the opposites** (as archetype E predicts): at ε=0.8 the
fine clustering produced exactly 2 clusters —
`{taxes: raise + no-raise}` (10) and `{peace: make + no-make}` (10). The split
is entirely the LLM judge's doing.

**`score.mjs` verdict: ❌ FAIL — 7/11 checks — but every negation-specific
check passes.** The cosine step merged each pro+anti pair into one cluster; the
two-tier judge then **split each into a pro synth and an anti synth**
(`refinedFromDissent=2`), all four 100% stance-pure with zero contamination, and
the two subjects grouped into their correct topics. The 4 failures are the same
medoid-anchored recall tax seen elsewhere: `no-peace` (3/5) and `no-raise-taxes`
(4/5) shed 3 members total to dissent that did not re-form a clique. No member
was ever placed in the *wrong* stance.

## 5. Verdict

**PASS on the property under test: negation/opposites are NOT merged.** A path
that clustered by cosine alone would have returned 2 synths (one per subject,
pro+anti conflated); the production two-tier judge returned 4 stance-pure synths.
This is the archetype-E behaviour the design protocol says requires the LLM
verifier, and it works. The overall `score.mjs` FAIL is purely recall
(3 of 20 negation-side members dropped), not a stance error — consistent with
the medoid-anchored recall limit documented in the loose-corpus run.

## 5b. Re-run on the quorum-tolerant judge (commit `1ee58ecd5`)

The 3-member recall loss above was traced to `refineComponent`'s **strict
complete-linkage** rule: a genuine member shed to dissent could not re-join its
clique because one stray "different" pair-verdict violated unanimity, even though
within-group cosine was 0.91–0.95. Commit `1ee58ecd5` relaxes the join rule to
**quorum-tolerant linkage** — a candidate joins if it is "same" with a quorum
(`ceil(0.75·cliqueSize)`, capped at one absolute dissent) of the current members
— which rescues single-noisy-verdict drops while keeping a genuinely-opposite
member (different against the *whole* opposing group) out, so anti-chaining is
preserved. Zero added LLM cost: the quorum check reads only internal pair
verdicts already fetched.

Re-running the **same input vectors** (seeded directly from this run's
`statements.json` + `embeddings.json` via
[`functions/scripts/seedNegationFromArtifacts.ts`](../../../functions/scripts/seedNegationFromArtifacts.ts),
no embedding API needed) through the production two-tier judge, **3× for
stability**, gives an identical result every run:

| Metric | original (7/11) | quorum-tolerant re-run | Pass |
|---|---|---|---|
| Synth count | 4 | **4** | ✅ |
| `make-peace` / `raise-taxes` | 5 / 5 | 5 / 5 | ✅ |
| `no-peace` / `no-raise-taxes` | 3 / 4 | **5 / 5** | ✅ |
| Options assigned | 17 | **20** | ✅ |
| Cross-stance contamination | 0 | **0** | ✅ |
| Topic-clusters | 2 | **2** | ✅ |

`twoTierJudge` stats per run: `llmCalls=30`, `refinedFromDissent=2`, `dropped=0`.
This takes the negation run from **7/11 (PASS-property, recall-failed)** to a
full pass: all four stance-synths recovered at size 5, opposites still never
merged. The medoid-anchored recall tax that this run exposed is the bug the
quorum fix closes.

## 6. Notes & reproduction

- ε=0.8 used (not the 0.45 of the other runs) because the small, semantically
  tight working set forms no cluster below it — recorded as a parameter finding.
- The two-tier split is LLM-driven, so member counts on the dropped side can
  vary run-to-run; the *stance separation* (4 pure synths) is the stable result.
