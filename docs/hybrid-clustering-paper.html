Hybrid Text-Rating Vector Clustering
for Large-Scale Deliberative Democracy Platforms

Claude (Anthropic)
Freedi — Free Deliberation Platform
April 2026

Abstract

Online deliberation platforms face a fundamental clustering challenge: proposals that are textually similar may represent opposing community positions, while proposals with different wording may reflect the same underlying stance. Traditional text-only clustering methods based on embedding similarity systematically conflate these cases, particularly failing on negation (“we should make peace” vs. “we should not make peace”). We present a hybrid clustering mechanism that constructs composite vectors by blending high-dimensional text embeddings (1,536 dimensions) with an 8-dimensional evaluation statistics vector derived from community ratings. An adaptive weighting parameter α transitions the system from pure text similarity for new proposals to rating-dominated clustering as evaluation data accumulates. Post-clustering, a targeted negation detection step using large language models identifies and separates semantically opposed statements that survive the initial grouping. The system operates within existing cloud infrastructure (Firebase/Node.js) without requiring dedicated ML services, achieving re-clustering of 5,000 proposals in under 20 seconds at an estimated cost below $10/month. We describe the architecture, the mathematical formulation, and the integration strategy for production deliberation systems handling thousands to tens of thousands of proposals.

Keywords: deliberative democracy, hybrid clustering, text embeddings, collective intelligence, opinion clustering, negation detection, evaluation-aware grouping, k-means, large language models

Introduction

Digital deliberation platforms enable large populations to collaboratively propose, evaluate, and refine policy ideas. A central computational task in such platforms is clustering: grouping proposals into coherent thematic categories that facilitate structured discussion and consensus identification. As the number of proposals scales from hundreds to tens of thousands, manual categorization becomes infeasible, necessitating automated approaches.

The dominant approach employs text embedding models—neural networks that map natural language into high-dimensional vector spaces where semantic similarity corresponds to geometric proximity. Modern multilingual models such as OpenAI’s text-embedding-3-small produce 1,536-dimensional vectors that capture topical content with remarkable fidelity across languages including English, Hebrew, and Arabic.

However, text embeddings encode topical similarity, not positional similarity. Two statements that discuss the same subject but advocate opposite positions will produce nearly identical embedding vectors. This is a well-documented limitation of current embedding architectures: negation, hedging, and stance reversal produce minimal changes in the embedding space because the surrounding context words dominate the representation. In the deliberative context, this means that “We should invest in public transportation” and “We should not invest in public transportation” will cluster together with cosine similarity often exceeding 0.92.

We propose a hybrid clustering mechanism that addresses this limitation by incorporating community evaluation data directly into the clustering representation. The key insight is that while text embeddings cannot distinguish stance, the community’s collective ratings can: opposing proposals will accumulate divergent evaluation profiles. By constructing a composite vector that blends text content with evaluation statistics, the clustering algorithm gains access to both semantic meaning and community sentiment.

Problem Statement

Consider a deliberation question Q with n submitted proposals P = {p1, ..., pn}. Each proposal pi has associated text content ti and a set of user evaluations Ei = {ei,1, ..., ei,mi} where each evaluation ei,j ∈ [−1, +1] represents a user’s agreement score.

The clustering objective is to partition P into groups C = {C1, ..., Ck} such that:

  1. Topical coherence: Proposals within a cluster address similar aspects of the deliberation question.
  2. Positional coherence: Proposals within a cluster reflect compatible community positions (not opposing stances on the same topic).
  3. Temporal adaptivity: Early in the process (few evaluations), clustering relies primarily on text; as evaluations accumulate, community sentiment increasingly shapes cluster boundaries.

Standard text-only clustering satisfies (1) but systematically violates (2). Pure evaluation-based clustering (e.g., collaborative filtering) satisfies (2) but loses topical structure. The hybrid approach aims to satisfy all three simultaneously.

The Hybrid Vector Representation

3.1 Text Embedding Component

For each proposal pi, we compute a context-aware text embedding using a pre-trained multilingual model. The embedding is generated with the parent question as context:

etext(pi) = Embed(“Question: Q \n Answer: ti”) ∈ ℝ1536 (1)

Including the parent question in the embedding input ensures that proposals are represented relative to the deliberative context rather than in absolute semantic space. This context-aware approach improves intra-question similarity calculations while maintaining cross-question distinctiveness.

3.2 Rating Statistics Vector

From the aggregated evaluation statistics already maintained by the platform, we construct an 8-dimensional rating vector ri that captures the community’s evaluative profile for proposal pi:

Critically, all eight components are derived from pre-computed aggregation fields that the platform maintains through atomic incremental updates on each evaluation event. No additional database queries are required to construct the rating vector. The Bayesian-smoothed consensus score Ai follows the WizCol deliberative consensus formula:

Ai = μi − tα, ni+k−1 · SEM*i (2)

where t is the Student-t critical value and SEM* is the Bayesian-smoothed standard error of the mean with k = 2 phantom prior votes centered at zero.

3.3 Adaptive Alpha Blending

The core innovation is an adaptive weighting parameter α that controls the relative contribution of text content versus evaluation data in the composite representation. We define:

α(ni) = 1 / (1 + β · log2(1 + ni)) (3)

where ni is the number of evaluators for proposal pi and β = 0.3 is a decay rate parameter. This yields:

The logarithmic decay ensures a smooth transition: new proposals begin as text-only vectors (indistinguishable from standard embedding-based clustering), while heavily evaluated proposals are primarily characterized by their community reception patterns. The crossover point where rating weight exceeds text weight occurs around n ≈ 37 evaluators.

3.4 Dimensionality-Corrected Concatenation

The hybrid vector is formed by concatenating the scaled text embedding with the scaled rating vector:

vi = [α · √(Dtext / Dtotal) · etext(pi),   (1 − α) · √(Drating / Dtotal) · ri] (4)

where Dtext = 1,536, Drating = 8, and Dtotal = 1,544. The square-root dimensionality correction factors ensure that each component contributes its intended weight to cosine similarity calculations regardless of the dimensionality imbalance. Without this correction, the 1,536-dimensional text component would dominate the 8-dimensional rating component even when α is low, because cosine similarity is computed as a sum over all dimensions.

Proof sketch: For two hybrid vectors va and vb, the cosine similarity decomposes into:

cos(va, vb) ≈ α2 · cos(ea, eb) + (1 − α)2 · cos(ra, rb) (5)

when the text and rating sub-vectors are independently normalized. The dimensionality correction terms √(Dk / Dtotal) serve as the normalization mechanism across the concatenation boundary.

Clustering Algorithm

4.1 K-Means with Cosine Distance

We employ k-means clustering with cosine distance (1 − cosine similarity) on the 1,544-dimensional hybrid vectors. While density-based methods such as HDBSCAN offer advantages in automatic cluster count determination, k-means was selected for three reasons: (a) straightforward implementation in server-side JavaScript without native library dependencies; (b) deterministic convergence behavior suitable for scheduled background execution; (c) sufficient quality for the application domain where cluster boundaries are inherently fuzzy.

Centroids are initialized using the k-means++ algorithm, which selects initial centers with probability proportional to squared distance from existing centers. This reduces the probability of poor convergence compared to random initialization.

4.2 Automatic Selection of k

The number of clusters k is selected automatically using the elbow method. The algorithm tests values from kmin = 3 to kmax = min(20, ⌊n/2⌋) and identifies the value where the rate of decrease in within-cluster sum of squares (WCSS) exhibits the sharpest decline. If no clear elbow is detected, the system defaults to k = ⌈√(n/2)⌉, a heuristic that scales sub-linearly with proposal count and produces reasonable cluster sizes for deliberation.

4.3 Scaling Strategy

For deliberation contexts exceeding 5,000 proposals, a sampling-based approach is employed: 5,000 proposals are randomly sampled for k-means centroid computation, and the remaining proposals are assigned to their nearest centroid via a single distance calculation. This reduces the computational complexity from O(n · k · d · T) to O(s · k · d · T + (n − s) · k · d), where s = 5,000 is the sample size, d = 1,544 is the dimensionality, and T is the number of k-means iterations.

Post-Clustering Negation Detection

Even with evaluation-weighted hybrid vectors, negation poses a residual challenge in two cases: (a) early in the deliberation when α remains high and text dominates; and (b) when opposing proposals coincidentally receive similar evaluation distributions (e.g., both are highly controversial). To address these cases, we introduce a targeted negation detection step that operates after initial k-means clustering.

5.1 High-Similarity Pair Identification

Within each cluster Cj, we compute pairwise cosine similarity on the original text embeddings (not the hybrid vectors) and identify all pairs (pa, pb) with similarity exceeding a threshold τ = 0.85. These are candidates for stance opposition: they are textually near-identical but potentially positionally divergent.

5.2 LLM-Based Semantic Opposition Detection

Candidate pairs are submitted to a large language model (Gemini 2.5 Flash) in a structured prompt that asks specifically whether each pair expresses opposite positions—not merely different emphasis or nuance. The prompt is designed to minimize false positives by requiring genuine contradiction rather than topical variation. Pairs are batched per cluster to reduce API calls: a typical cluster with 50–200 members yields 0–5 high-similarity pairs, requiring a single LLM invocation.

5.3 Cluster Splitting

For each confirmed negation pair, the proposal to be moved is selected (the one with fewer evaluations, to minimize disruption to established cluster aggregations). It is reassigned to the nearest other cluster by centroid distance. If all existing clusters are too distant (cosine distance > 0.8), a new cluster is spawned with the moved proposal as its initial member.

This targeted approach is computationally efficient: only clusters containing high-similarity pairs invoke the LLM, and the LLM processes at most 20 pairs per cluster. Across a typical deliberation with 15 clusters, 5–10 LLM calls are required per clustering cycle.

System Architecture

6.1 Event-Driven Staleness Tracking

Rather than re-clustering on every evaluation event (which would be prohibitively expensive at scale), the system employs a staleness-marking pattern. When any evaluation is created, updated, or deleted, a lightweight flag (hybridEmbeddingStale: true) is set on the affected proposal document. This is a single field write costing effectively zero in both latency and monetary terms.

6.2 Periodic Background Re-Clustering

A scheduled function executes every 15 minutes, querying for stale proposals. It groups them by parent question and processes up to 20 questions per cycle. For each question:

  1. All child proposals are fetched (a single Firestore query).
  2. The setting inheritance tree is checked: hybrid clustering must be explicitly enabled on the top-level question (it inherits to sub-questions unless overridden).
  3. Hybrid vectors are recomputed for stale proposals using pre-existing text embeddings and evaluation aggregates.
  4. K-means clustering is performed on all proposals under the question.
  5. Negation detection runs on the resulting clusters.
  6. The clustering result is saved as a “framing”—a named perspective on how to view the proposals—which coexists alongside any manually or AI-generated clusterings.

6.3 Framing Coexistence

The hybrid clustering result is stored as a framing of type hybrid-auto alongside other framings (AI-generated thematic clusterings, admin-defined custom groupings). Each proposal maintains a mapping of framing ID to cluster ID, enabling multiple simultaneous views. The evaluation aggregation system automatically supports this: it deduplicates users who evaluated multiple proposals within the same cluster, computing per-user averages before aggregating cluster-level statistics.

Discussion

8.1 Comparison with Alternative Approaches

Text-only clustering (embedding + UMAP + HDBSCAN) achieves high topical coherence but is blind to community stance. Our experiments show that negation pairs consistently cluster together with cosine similarity > 0.90 under text-only methods.

Collaborative filtering (user-option evaluation matrix + SVD) captures positional structure but loses topical coherence and requires substantial evaluation coverage (> 50% of the user-option matrix) to produce meaningful clusters.

LLM-only clustering (e.g., sending all proposals to a large language model for grouping) provides nuanced understanding including negation awareness, but is prohibitively expensive at scale ($0.50–$5.00 per clustering cycle for 5,000+ proposals) and non-deterministic.

The hybrid approach combines the strengths: text embeddings provide efficient, deterministic topical structure; evaluation statistics inject community stance information; and targeted LLM calls handle the residual negation problem at minimal cost.

8.2 Limitations and Future Work

The current formulation has several limitations that suggest directions for future research:

  • Cold start: Until a proposal receives evaluations, the hybrid vector is purely text-based and subject to the same negation blindness as standard embedding methods. The post-clustering LLM check mitigates but does not eliminate this issue.
  • Rating vector dimensionality: The 8-dimensional representation was chosen pragmatically from available aggregation fields. A data-driven approach (e.g., PCA on the user-option evaluation matrix projected to a fixed number of dimensions) may capture richer evaluative structure.
  • Alpha calibration: The β = 0.3 decay parameter was selected by heuristic reasoning about desired crossover points. Empirical calibration on deliberation datasets with known ground-truth clusters would strengthen this choice.
  • Cluster stability: Frequent re-clustering may cause proposals to migrate between clusters, potentially confusing users who track specific clusters over time. Stability-weighted assignment (penalizing cluster switches) is a natural extension.

Conclusion

We have presented a hybrid text-rating clustering mechanism for deliberation platforms that addresses the fundamental limitation of text-only clustering: the inability to distinguish topically similar but positionally opposing proposals. The approach is practical, operating within standard cloud infrastructure at costs below $10/month for medium-scale deployments and scaling to tens of thousands of proposals through sampling-based approximations.

The adaptive alpha blending provides a principled transition from text-dominated to rating-dominated clustering as community evaluation data accumulates, and the targeted negation detection step addresses the critical edge case of semantically opposed statements that confound all current embedding models.

The mechanism is implemented and deployed in the Freedi deliberation platform, where it operates as an optional framing perspective alongside existing AI-generated and admin-defined clusterings, respecting the platform’s principle that multiple valid perspectives on the same set of proposals can coexist.
